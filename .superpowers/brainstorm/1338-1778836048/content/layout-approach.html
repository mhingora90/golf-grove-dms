CRM Layout for 100s of Leads

Current kanban loads all leads at once — no pagination, no filtering, no sort. Which approach scales best for your team?

A

Smart Kanban (enhanced current)

Keep the kanban but add: per-column pagination (show 20, load more), server-side search, filter bar (source / assigned-to / date), sort within columns. Familiar workflow, no relearning.

Pros

  • No workflow change
  • Drag-to-move stage still possible
  • Animations stay

Cons

  • 7 columns still scroll awkwardly on laptops
  • Hard to compare leads across stages
B

List + Kanban Toggle

Two views — a paginated sortable table/list view (default for power users, bulk actions) and the existing kanban for visual pipeline overview. Toggle between them with a button. Server-side pagination + filters in both.

Pros

  • Table = scan 50 leads at once
  • Kanban = stage overview
  • Best of both worlds

Cons

  • More UI to build
  • Two UIs to maintain
C

Pipeline List (replace kanban)

Single paginated list view. Stage shown as a colored badge column. Filter/sort/search server-side. Compact rows — see 30+ leads at a glance. Stage change via inline dropdown. Kanban removed.

Pros

  • Simplest to build
  • Scales to any number of leads
  • Easy bulk-stage updates

Cons

  • Loses pipeline visual feel
  • No drag-and-drop
